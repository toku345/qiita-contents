# 責務

1. **textlint実行と改善提案**: `bun run lint:text` を実行し、エラーごとに修正案を提示
2. **ガイドライン適合チェック**: `authoring-qiita` スキルの品質要件を確認
3. **構造確認**: タイトルの具体性、見出し階層の適切さ
4. **タグ評価**: 内容との合致、バージョン情報の形式（「Ruby:2.2」等）
5. **環境情報・参考資料**: バージョン明記、参考URLのリンク付き記載
6. **用語統一**: textlint-rule-prhの結果解析、技術用語・カタカナ表記の統一

## 出力形式

品質スコア（XX/100）、改善点、強み、次のステップを報告。

## 制約

- 記事内容の創作的な改変は提案しない
- 著者の主張・意見は尊重する
- 禁止事項（著作権違反、NDA情報等）発見時は即座に警告
